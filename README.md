# PPT2MD-V10 (Parallel Brain & Stream Fixed) 🚀

> 基于 **Qwen3-VL-Plus** (视觉) + **Qwen-Plus** (大脑) 的双引擎 PPT 转 Markdown 工具。
> V10 版本引入了 **全并行架构**，彻底打破了传统“上下文依赖导致必须串行”的限制，实现极速生成。

## 📖 简介

本项目旨在将大量的课件/论文/会议 PPT 图片批量转换为结构清晰的 Markdown 笔记。

**V10 版本的重大突破：**
传统的 PPT 转 Markdown 工具为了保持上下文连贯（如“如图所示”指代上一页），必须一页页串行处理。
**V10 采用了“两阶段 + 滑动窗口”架构：**
1.  **Stage 1 (视觉提取)**：使用 `Qwen3-VL-Plus` 并发提取所有页面的 Raw Data（OCR + 几何特征描述）。
2.  **Stage 2 (逻辑重组)**：使用 `Qwen-Plus` 文本模型，配合 **5页滑动窗口** (Prev-2, Prev-1, Target, Next-1, Next-2) 并行重组 Markdown。

这意味着：**你可以同时开启 60+ 个线程并发处理同一个 PPT，速度提升 10 倍以上！**

## ✨ 核心特性 (V10)

*   **🚀 双引擎架构**：
    *   **视觉引擎 (Vision)**：`qwen3-vl-plus` 专注 OCR 和 TikZ 几何特征提取。
    *   **大脑引擎 (Brain)**：`qwen-plus` 专注逻辑重组、公式修正和上下文润色。
*   **⚡ 极致并发 (Parallel Brain)**：
    *   不再受限于上下文串行依赖。通过预读取 Raw Data，Stage 2 可以火力全开。
    *   默认配置：**60 视觉并发 + 60 大脑并发**。
*   **🧠 5页滑动窗口 (Sliding Window)**：
    *   Brain 模型在处理第 N 页时，能同时看到 N-2 到 N+2 页的 Raw Data。
    *   完美解决符号一致性问题（如前一页定义了 $v$，这一页不会误识别为 $\nu$）。
*   **🎨 TikZ-Ready 图形分析**：
    *   专门针对架构图、流程图优化，输出详细的节点、连线、布局描述，为后续生成 TikZ 代码做准备。
*   **💻 沉浸式终端 UI**：
    *   基于 `rich` 的实时仪表盘，显示双阶段进度、速度、ETA。
*   **🛡️ 鲁棒的流式处理**：
    *   内置重试机制和流式输出解析，有效应对 API 波动。

## ⚙️ 性能调优与限流分析 (Technical Detail)

本项目默认参数经过严格的数学推导，旨在平衡**生成速度**与**API 限流风险**。

### 1. 默认配置 (V10)

```python
# 全局并发控制 (同时处理多少个 PPT 文件夹)
MAX_PPT_WORKERS = 1

# Step 1 视觉提取并发数 (默认 60 - 激进模式)
VISION_BATCH_WORKERS = 60  

# Step 2 逻辑重组并发数 (默认 60 - 激进模式)
BRAIN_BATCH_WORKERS = 60 
```

### 2. 为什么默认并发设为 60？(与限流的博弈)

V10 版本采用了**全并行架构**，理论上可以无限并发。默认设置 `60` 是为了在不触发大规模 429 错误的前提下，尽可能跑满 API 带宽。

**阿里云官方 Qwen3-VL / Qwen-Plus 限流规则 (参考)：**

| 模型 | RPM (每分钟请求) | TPM (每分钟 Token) |
| :--- | :--- | :--- |
| **qwen3-vl-plus** | 1,200 | 1,000,000 |
| **qwen-plus** | 15,000 | 5,000,000 |

> 注：实际限流可能按秒级 (TPS) 计算，且与账号等级有关。

#### Token 计算公式 (Qwen3-VL)

根据官方文档，**Qwen3-VL** 系列模型的图像 Token 计算方式如下（每个 Token 对应 32x32 像素）：

$$ Token_{img} = \frac{H_{bar} \times W_{bar}}{32 \times 32} + 2 $$

其中 $H_{bar}, W_{bar}$ 是经过预处理缩放后的尺寸。
简单估算：一张 1080P ($1920 \times 1080$) 图片约为 **2,000 - 2,600 Tokens**。

#### 计费与成本预估

本项目内置了**成本预估功能**。在任务开始前，会自动扫描图片尺寸，根据阿里云阶梯计费规则估算本次任务的 Token 消耗与费用。

**参考价格 (Qwen3-VL-Plus):**
*   输入: 0.0015 元/千Token (32k-128k档)
*   输出: 0.015 元/千Token

**参考价格 (Qwen-Plus):**
*   输入: 0.0008 元/千Token
*   输出: 0.002 元/千Token

**结论**：
*   **默认 (60)**：追求极致速度，依赖重试机制抵抗限流。
*   **保守 (3-5)**：如果你的账号频繁报错或等级较低，请手动修改 `ppt2md.py` 中的并发参数。

## ❓ 常见问题 (FAQ)

**Q: 为什么 V10 比旧版本快这么多？**
A: 旧版本必须等第 1 页生成完才能生成第 2 页（为了上下文）。V10 先并发提取所有页面的 Raw Data，然后利用“滑动窗口”技术，让第 2 页在生成时直接读取第 1 页的 Raw Data，从而实现了完全并行。

**Q: 消耗 Token 会变多吗？**
A: 会略微增加。因为 Stage 2 的 Prompt 中包含了前后各 2 页的 Raw Data 作为上下文。但考虑到 `Qwen-Plus` (文本模型) 的价格远低于视觉模型，总体成本增加可控，换来的是巨大的速度提升和更好的符号一致性。

**Q: 任务中断了怎么办？**
A: 直接重新运行。程序会检测 `markdown_output` 下已生成的文件和 `temp_raw_vision` 下的中间文件，自动跳过已完成的部分。

**Q: 如何查看详细报错？**
A: 请查看 `log/` 文件夹下的日志文件。
