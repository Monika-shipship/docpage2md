## API 配额规则及错误码说明

[English](https://ai.baidu.com/ai-doc/AISTUDIO/pmjcld5qm)

## 1\. API 调用配额与限制规则

### 每日调用配额

- **总量限制** ：每个用户针对每个模型，每日可解析的页面总数上限为 3000 页。
- **超额反馈** ：当当日解析量超过 3000 页时，后续请求将无法处理，系统会返回 429 (Too Many Requests) 错误码。

### 单文件处理限制

- **页数建议** ：为避免处理时间过长导致网关超时，建议单个文件的页数 **不超过 100 页** 。
- **超页处理** ：如果上传的文件超过 100 页，API 将 **仅解析前 100 页的内容** ，后续页面将被忽略。

### 每日配额不足 (3000页上限) 解决方案

如果您有更高的页面解析需求，请填写 [问卷](https://paddle.wjx.cn/vm/mePnNLR.aspx#) 免费申请提升配额。

## 2\. 错误码说明

| 错误码 | 说明 | 解决建议 |
| --- | --- | --- |
| 403 | Token 错误 | 检查 Token 是否正确，或 URL 是否与 Token 匹配 |
| 413 | 请求体过大 | 请减少 PDF 文件的页数或文件大小 |
| 422 | 参数无效 | 请参考 errorMsg 进行解决 |
| 429 | 超出单日解析最大页数 | 请使用其他模型或稍后再试 |
| 500 | 服务器内部错误 | 如果频繁遇到 500 问题，请联系 PaddleOCR 官方人员 |
| 503 | 当前请求过多 | 请稍后再试 |
| 504 | 网关超时 | 请稍后再试 |
