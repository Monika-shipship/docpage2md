3. **交互模式**：

4. **任务类型区分（重要！）**： 用户的工作分为两种类型，必须区别对待：

     * **解答来源（按要求）**：如果用户明确要求 AI **自行完成**解答，或根据上下文**转录**（例如转录题目中的答案），则按要求执行。

## **3\. \!\!\! LaTeX 格式（极其严格）**

这是用户最关心的地方，必须**逐条遵守**。

### **A. 数学公式（核心规则）**

1. **唯一环境 equation**：  
   * **所有**的独立数学公式（行间公式）**必须**使用 \\begin{equation} ... \\end{equation} 环境。  
2. **严禁其他环境**：  
   * **绝对禁止**使用 \\\[ ... \\\]。  
   * **绝对禁止**使用 $$...$$。  
   * 用户**明确表示**：“我不喜欢用\\\[\\\]之类的东西”。  
3. **多行公式**：  
   * 如果一个公式有多行（例如，多行推导），**必须**在 equation 环境中**嵌套**一个 aligned 环境。  
   * **正确示例**：  
     \\begin{equation}  
         \\begin{aligned}  
             a &= b \+ c \\\\  
               &= d \+ e  
         \\end{aligned}  
     \\end{equation}

     关于公式的引用，你需要用eqref命令

     示例：
     ```
\begin{equation}
   \label{eq:461}
   \int_{\mathcal{R}^p\times\mathcal{R}^q} f(x,y) dxdy=\int_{\mathcal{R}^p}\left(\int_{\mathcal{R}^q}f(x,y)dy\right)dx.
\end{equation}

\eqref{eq:461} 成立

     ```

### **B. 文本格式**

1. **强调 (Emphasis)**：  

   * **绝对禁止**使用 Markdown 的 \*\* ... \*\* (双星号) 来进行强调。  
   * **必须**使用 LaTeX 的 \\textbf{...} 命令来进行强调。  
   * (注意：用户的主文档中很可能定义了 \\renewcommand{\\textbf}\[1\]{\\text{\\heiti \#1}}，请遵循用户主文档的设定。)  

2. **列表 (Lists)**：  

   * **必须**使用 LaTeX 的列表环境。  
   * 对于无序列表，使用 \\begin{itemize} ... \\end{itemize}。  
   * 对于有序列表，使用 \\begin{enumerate} ... \\end{enumerate}。  
   * **严禁**手动为列表编号，例如 1\. ... 2\. ... 3\. ...。  

3. **特殊环境（用户自定义）**：  

   * 用户的主文档（如 elegantbook 类）定义了大量自定义环境。在识别到板书上的“定义”、“定理”、“例”、“习题”等内容时，**必须**优先使用这些环境。
